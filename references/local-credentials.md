# 本机凭据文件

在 macOS 上，默认使用本机凭据文件避免把 SSH 密码发送到对话。

## 创建与填写

DNS 核验完成后，先收集非敏感信息：VPS IPv4、SSH 端口、SSH 用户名、系统、面板域名。基于面板域名生成仅含字母、数字、`.`、`_`、`-` 的凭据名称，然后自动运行：

```bash
scripts/create-local-credentials.sh <名称> <IPv4> <端口> <用户名> <系统> <面板域名>
```

脚本会创建 `~/.config/3xui-vps-deploy/<名称>.env`，目录权限为 `700`、文件权限为 `600`，并用 TextEdit 打开。绝不覆盖已存在文件。

用户只填写 `SSH_PASSWORD=` 后的临时 root 密码，保存并关闭 TextEdit 后回复“已保存”。不要要求用户在聊天中发送密码。

## 安全读取

确认文件权限为 `600`，并在不打印内容的前提下检查六个字段均非空。仅在执行 SSH 时将密码传给 SSH 进程；不要 `cat` 文件到终端输出、不要把密码放入命令参数、不要把文件加入 Git、同步盘或最终报告。

部署完成后提醒用户在商家后台重置 root 密码，并更新或删除该凭据文件。

## 兼容模式

非 macOS、无法打开 TextEdit、或用户明确要求时，才直接收集临时 SSH 密码。仍不得把密码写入文件、日志或最终报告。
